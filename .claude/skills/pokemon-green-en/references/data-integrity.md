# 데이터 무결성 검증 가이드

**중요**: 게임 진행 시 반드시 데이터 파일을 참조하여 정확한 정보만 사용해야 합니다. 추측이나 기억에 의존하지 마세요.

## 필수 검증 항목

| 상황 | 참조 파일 | 검증 내용 |
|------|----------|----------|
| 기술 습득 | `data/pokemon/learnsets.json` | 해당 레벨에 배울 수 있는 기술인지 확인 |
| 진화 | `data/pokemon/evolutions.json` | 정확한 진화 레벨/조건 확인 |
| 야생 출현 | `data/world/encounters.json` | 해당 지역에 출현하는 포켓몬인지 확인 |
| 기술 사용 | `data/moves/moves.json` | 위력, 명중률, PP, 효과 확인 |
| 타입 상성 | `data/types/chart.json` | 데미지 배율 확인 |
| 트레이너 전투 | `data/world/trainers.json` | 트레이너 파티 구성 확인 |

## 검증 규칙

### 1. 기술 습득 시
- 반드시 `learnsets.json`을 Read 도구로 읽어서 확인
- 해당 포켓몬이 해당 레벨에 배울 수 있는 기술만 추가
- 기술머신/비전머신은 별도 조건 확인

### 2. 진화 시
- 반드시 `evolutions.json`을 Read 도구로 읽어서 확인
- 레벨 진화: 정확한 레벨 도달 시에만
- 돌 진화: 해당 진화석 사용 시에만
- 교환 진화: 연결의끈 또는 NPC 교환 시에만

### 3. 야생 포켓몬 출현 시
- 반드시 `encounters.json`을 Read 도구로 읽어서 확인
- 해당 지역에 등록된 포켓몬만 출현
- 출현율과 레벨 범위 준수

### 4. 전투 데미지 계산 시
- `moves.json`에서 기술 위력/타입 확인
- `chart.json`에서 타입 상성 배율 확인
- 1세대 공식 정확히 적용

## 사용자 검증 요청

사용자가 다음과 같이 요청하면 해당 데이터 파일을 직접 읽어서 검증:
- "데이터 확인해줘"
- "[포켓몬]이 [레벨]에 [기술] 배워?"
- "[장소]에 [포켓몬] 나와?"
- "[기술] 정보 알려줘"

## 오류 방지 체크리스트

게임 진행 중 다음 상황에서는 **반드시** 데이터 파일 참조:
- [ ] 새로운 기술 습득 메시지 출력 전
- [ ] 진화 연출 시작 전
- [ ] 야생 포켓몬 조우 시
- [ ] 트레이너 전투 시작 시
- [ ] 데미지 계산 시 (타입 상성)
- [ ] 포켓몬 스탯 표시 시
